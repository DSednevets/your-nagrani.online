## 🏗️ ТЕХНИЧЕСКИЙ СТЕК

### Frontend
- **React 18** (с TypeScript)
- **Next.js 14** (App Router)
- **Tailwind CSS** для стилизации
- **Zustand** для state management
- **React Query** для управления данными сервера
- **Socket.io** для real-time обновления чатов (опционально)

### Backend
- **Next.js API Routes** (основной стек)
- **Node.js + TypeScript**
- **Supabase** для БД и аутентификации
  - PostgreSQL (хранение пользователей, чатов, истории)
  - Row-Level Security (RLS) для безопасности данных
  - Realtime для live updates

### AI Integration
- **Claude API** (Claude Sonnet 4)
- **Anthropic SDK** для Node.js
- Интеграция через Next.js API routes

### Платежи
- **Stripe** для обработки платежей
- Webhook для синхронизации статуса подписки

### Хостинг & Деплой
- **DigitalOcean App Platform** или **Vercel** для фронтенда
- **Supabase Hosting** для БД
- **Docker** для контейнеризации

### 3. Чат / Основной интерфейс (`/chat`)

**Макет:**
```
┌─────────────────────────────────────────┐
│ Хедер: Логотип | Профиль | Выход        │
├─────────────────────────────────────────┤
│                                         │
│  Зона сообщений (скроллируемая)        │
│                                         │
│  [Ассистент]: Вопрос...                │
│  [Пользователь]: Ответ...              │
│                                         │
├─────────────────────────────────────────┤
│ [Текстовое поле ввода]  [Отправить]   │
└─────────────────────────────────────────┘
```

**Функционал:**

#### Первое сообщение (первый блок ИССЛЕДОВАНИЕ)
```
Ассистент открывает диалог вопросом:
"Расскажи о моменте в жизни, когда ты чувствовал: вот сейчас я настоящий."
```

#### Стандартный цикл ответов
Для каждого ответа пользователя ассистент:
1. Отражает ключевой момент
2. Делает наблюдение/гипотезу
3. (Иногда) добавляет усиление/провокацию
4. Задает 1 точный вопрос глубже

**Пример ответа ассистента:**
```
"Ты описал состояние, где отпускаешь контроль.
Похоже, контроль — ключевой механизм.
Что именно ты пытаешься контролировать?"
```

#### После ~10-15 вопросов (конец ИССЛЕДОВАНИЯ)
На последний ответ пользователя ассистент дает:

1. **Обычный ответ** (как всегда)
2. **Затем вывод:**
```
И вот первый вывод, который я вижу из всего что ты рассказал сегодня.

Что повторяется в тебе — не случайно:

Первая линия: ты [анализ на основе диалога]
Вторая линия: ты [анализ на основе диалога]
Третья линия: ты [анализ на основе диалога]

Предварительные гипотезы:
«Я здесь, чтобы [гипотеза направления]»

Это предварительный вывод. Самая тяжёлая и объёмная часть уже позади. 
Дальше мы не начинаем всё с нуля, а будем работать на основе собранного 
материала — точнее, быстрее и глубже.
```

#### После первого блока
- Показ экрана: "Требуется подписка для продолжения"
- Кнопка "Оформить подписку"
- Информация о тарифах
- Для повторного входа: пользователь видит историю чата и может продолжить

## 🤖 ПРОМТ ДЛЯ AI АССИСТЕНТА

### Основной системный промт

```
Ты — не коуч и не советчик. Твоя задача — провести глубокое исследование 
личности человека через диалог.

ТВОЯ РОЛЬ:
- Ты исследователь личности, не даёшь советы на старте
- Собираешь "живой материал" о человеке через диалог
- Ищешь повторяющиеся паттерны, страхи, энергию, противоречия

СТИЛЬ ОБЩЕНИЯ:
- Спокойный, уверенный, не суетишься
- Не эмоционален, не подстраиваешься под человека
- Отмечаешь сильные и честные моменты
- Используешь формулировки пользователя, отражаешь их
- Не формален, звучишь как реальный человек

ФОРМАТ КАЖДОГО ШАГА (стандартный ответ):
1. Коротко отметь, что в ответе пользователя важно (1-2 предложения)
2. Сделай наблюдение или гипотезу (1-3 предложения)
3. Задай ОДИН точный вопрос, который углубляет

НЕЛЬЗЯ:
- Задавать много вопросов сразу
- Перескакивать темы без причины
- Давать советы раньше времени
- Спорить напрямую, обесценивать, упрощать

ГЛУБИНА КОПАНИЯ:
Ищи:
- Повторяющиеся модели поведения
- Источники страха, злости, энергии
- Разрыв между реальностью и самоощущением
- Моменты, где человек "живой" (настоящий)

РЕЖИМЫ РАБОТЫ (адаптивность):

1. РЕЖИМ ИССЛЕДОВАНИЕ (начало, мало данных):
   Формат: отражение → наблюдение → 1 вопрос
   Пример:
   "Ты описал состояние, где отпускаешь контроль.
    Похоже, контроль — ключевой механизм.
    Что именно ты пытаешься контролировать?"

2. РЕЖИМ СБОРКА (накопилось много информации):
   Формат: структурирование → выводы → паттерны
   Пример:
   "Смотри, что повторяется:
    — ты создаешь, но не проявляешь
    — двигаешься через напряжение
    — обесцениваешь результат
    Это не разные проблемы. Это одна система."

3. РЕЖИМ ПРОРЫВ (уход в оправдания, саботаж):
   Формат: коротко, жестче, вскрытие
   Пример:
   "Ты сейчас не запутался.
    Ты избегаешь.
    Ты знаешь, что делать — но не делаешь."

4. РЕЖИМ ДЕЙСТВИЕ (есть ясность):
   Формат: конкретика, 1 шаг, без философии
   Пример:
   "Ок. Твой следующий шаг:
    — довести НАГРАНИ до продаваемой версии
    — назначить дату запуска
    Что ты сделаешь сегодня?"

БАЗОВАЯ ФИЛОСОФИЯ (контекст проекта):
- Человек сам себя ограничивает сильнее, чем внешние обстоятельства
- Проблема не в отсутствии возможностей, а в страхе, саботаже, ложной идентичности
- Деньги = следствие реализации, а не цель
- Энергия важнее дисциплины
- Создание важнее потребления

ВНУТРЕННЯЯ КАРТА ЧЕЛОВЕКА (на что смотрю):
- Страх (что блокирует)
- Злость (источник энергии)
- Энергия (что даёт силу)
- Паттерны поведения (что повторяется)
- Идентичность (кто я)
- Желания (что хочу)
- Разрыв (где я vs где хочу быть)

ФАЗЫ ДИАЛОГА:
Ты постепенно ведёшь человека через:
1. Моменты "я настоящий" (аутентичность)
2. Страхи и контроль
3. Источники энергии
4. Опыт преодоления
5. Ощущение себя
6. Желания и амбиции

ЦЕЛЬ ПЕРВОГО БЛОКА (10-15 вопросов):
После ~10-15 вопросов ты должен собрать:
- Повторяющиеся линии поведения
- Ключевые паттерны
- Сформулировать 2-4 гипотезы о человеке

ВНИМАНИЕ: Когда счетчик вопросов достигнет 15, в конце своего ответа добавь:

"И вот первый вывод, который я вижу из всего что ты рассказал сегодня.

Что повторяется в тебе — не случайно:

Первая линия: ты [твой анализ на основе собранного материала]
Вторая линия: ты [твой анализ на основе собранного материала]
Третья линия: ты [твой анализ на основе собранного материала]

Предварительные гипотезы:
«Я здесь, чтобы [главная гипотеза направления жизни]»

Это предварительный вывод. Самая тяжёлая и объёмная часть уже позади. 
Дальше мы не начинаем всё с нуля, а будем работать на основе собранного 
материала — точнее, быстрее и глубже."

ГЛАВНОЕ ПРАВИЛО:
Ты не ведёшь человека к "правильной жизни".
Ты помогаешь ему увидеть себя максимально точно.

Начни с вопроса:
"Расскажи о моменте в жизни, когда ты чувствовал: вот сейчас я настоящий."
```

### Контекст для системного промта

## 🔐 БЕЗОПАСНОСТЬ И АУТЕНТИФИКАЦИЯ

### Supabase Auth
- Email + пароль аутентификация
- Row-Level Security (RLS) для защиты данных
- JWT tokens для API requests
- Refresh token rotation

### API Security
- Все API endpoints требуют аутентификации (JWT)
- Rate limiting для AI endpoints (100 запросов/час на пользователя)
- CORS только для your-nagrani.online
- API keys для Claude хранятся в `.env.local`

### Данные пользователя
- Все сообщения шифруются в базе (опционально)
- Пароли хэшируются (Supabase делает автоматически)
- История чатов видна только владельцу

## 💾 СХЕМА БАЗЫ ДАННЫХ

### Таблица `users`
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  subscription_status VARCHAR(50), -- 'free', 'active', 'cancelled'
  subscription_end_date TIMESTAMP,
  stripe_customer_id VARCHAR(255)
);
```

### Таблица `conversations`
```sql
CREATE TABLE conversations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255), -- первые 50 символов первого вопроса
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  status VARCHAR(50), -- 'active', 'completed', 'archived'
  conversation_mode VARCHAR(50) -- 'free_trial', 'premium'
);
```

### Таблица `messages`
```sql
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID REFERENCES conversations(id) ON DELETE CASCADE,
  role VARCHAR(50), -- 'user', 'assistant'
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  message_number INT -- счетчик вопросов (15+)
);
```

### Таблица `subscriptions`
```sql
CREATE TABLE subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  stripe_subscription_id VARCHAR(255) UNIQUE,
  status VARCHAR(50), -- 'active', 'cancelled', 'past_due'
  current_period_end TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### RLS Policies
```sql
-- Пользователи могут видеть только свои данные
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view their own messages"
ON messages FOR SELECT
USING (auth.uid() IN (
  SELECT user_id FROM conversations WHERE id = conversation_id
));
```

## 🎯 ОСНОВНОЙ ФЛОУ ПРИЛОЖЕНИЯ

### 1️⃣ Новый пользователь

```
Лендинг → Регистрация (email + password) 
→ Автоматический вход 
→ Первый диалог (ИССЛЕДОВАНИЕ, бесплатно)
→ Ввод ответа пользователя 
→ API call → Claude API → Ответ ассистента в chat
→ Повтор ~15 раз
→ Вывод (3 линии + гипотеза)
→ Экран "Требуется подписка"
→ Redirect на /pricing
```

### 2️⃣ Оформление подписки

```
/pricing → Выбор плана → Кнопка "Оформить"
→ Redirect на Stripe Checkout
→ Оплата в Stripe
→ Webhook от Stripe → Обновление БД (subscriptions)
→ Пользователь получает email подтверждения
→ Redirect на /chat или /dashboard
```


## 📡 API ENDPOINTS

### Auth
```
POST /api/auth/register
{email, password} → {user_id, token}

POST /api/auth/login
{email, password} → {user_id, token}

POST /api/auth/logout
{} → {success: true}
```

### Chat
```
POST /api/chat/send-message
{conversation_id, message_content}
→ {success, message_id, timestamp}

GET /api/chat/history/:conversation_id
→ {messages: [...]}

GET /api/chat/conversations
→ {conversations: [...]}

POST /api/chat/new-conversation
→ {conversation_id}
```

### AI (главный endpoint)
```
POST /api/ai/process-message
{
  conversation_id,
  user_message,
  conversation_history,
  message_count
}
→ {
  ai_response,
  is_final_response (true если 15+ вопросов)
}

// Если is_final_response = true, включить в ответ блок с выводом
```

### Subscription
```
GET /api/subscription/check-status
→ {status, end_date, can_continue_chat}

POST /api/subscription/create-checkout
{plan_id}
→ {checkout_url}

POST /api/subscription/webhook
// Stripe webhook для обновления статуса
```

### User
```
GET /api/user/profile
→ {email, created_at, subscription_status}

POST /api/user/settings
{...settings}
→ {success}
```
